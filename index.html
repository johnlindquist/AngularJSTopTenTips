1. Keep Perspective


I've used a lot of frameworks and they're all great.


2. Don't miss the point

Most common question, "How to integrate jQuery plugin?"

3. Keep code Testable

Keep Model, Controller, View separate

  • Don't declare data in a Controller, extract to a Service
  • Don't manipulate DOM in controller, do that in Directives
  • Don't declare data in the DOM (e.g., ng-init)

4. Understand Promises

  • $http, $resource, etc
  • ngRoute resolving
  • $interval, $timeout

5. Don't Build One Huge Module

6. Learn ng-cloak

Users don't want to see {{this}}

ngBind also helps

7. Learn Your Build Tools

  • Grunt
  • Gulp
  • etc

I don't care which one you use, just don't switch in the middle.

  • ng-min
  • html2js
  • concat, uglify, recess, etc
  • transpilers (CoffeeScript, TypeScript, Sass, Less, Stylus, etc)

8. Embrace Community

Lots of stuff on Bower: http://bower.io/search/?q=angular

  • angular-bootstrap
  • angular-ui
  • angular-ui-utils
  • ng-upload
  • Restangular
  • Ionic

9. Use Protractor

Seriously, it's great.

It has zero impact on your project.

10. Follow Conventions

Developer turnover is a big problem, don't be "clever"

blog.angularjs.org

  • Style guide
  • Avoid $rootScope for sharing data, use Services
  • Use custom crazy event systems at your own risk
  • Rely on binding
